Unified Agent Kernel

AI 对话生图页面统一内核架构图

这版图对应新的 unified-agent-kernel 方案：把聊天 loop、图像生成任务系统、review、requires-action、 resume 与 cancel 收到同一个 AgentKernel 下，Surface 只做输入与投影订阅，Toolbox 负责统一调度和语义校验。

Diagram type: overview + runtime flow · Format: HTML+SVG · Style: reference-style

1. 总览图

左边是页面与用户交互层，中间是唯一执行真相所在的 Kernel，右边是 Toolbox 与 Provider。底部 persistence strip 持有写模型和投影缓存，任务卡、资产可见态、transcript 都从写模型派生。

Surface adapters Kernel ownership Toolbox / provider band Persistence + derived views

2. 核心运行时循环

这张图只画最关键的 loop：用户 turn 进入 Kernel，LLM 负责 planning，Toolbox 负责分类和执行，Kernel 决定是 reinject、 持久化 `AgentJob`，还是结束 turn。`interactive_tool`、`job_tool`、`kernel_step` 的处理语义明确分开。

这版图对应的关键设计点是：`TurnRuntimeState` 管当前 turn 的短生命周期执行态，`AgentJob` 管可恢复、可取消、 可继续的持久执行真相；`BackgroundTaskView`、可见资产、聊天 transcript 都是 projection，不再反向决定生命周期。
